# 需求分析技能

基于用户需求和现有代码逻辑，输出 `.workflow/requirement-analysis.md`。

## 输出要求

- 使用 `templates/requirement-analysis-template.md` 作为结构模板
- 明确区分：新增功能、改造现有功能、兼容性修改、配置调整、数据库变更
- 必须分析现有代码中是否已有相似实现
- 必须指出涉及模块、接口、表、缓存、消息、定时任务等影响面

## 分析步骤

1. 先把用户需求整理成明确的业务目标、边界和验收点。
2. 搜索现有模块、Controller、Service、Mapper、实体、常量、枚举中的相似逻辑。
3. 判断改动属于 Admin 还是 Web 链路，必要时同时分析两边。
4. 输出改动范围、风险点、是否涉及数据库和配置。
5. 将结论同步到 `.workflow/change-summary.json` 的基础字段。

## 关键要求

- 不要只复述用户需求，要结合现有代码逻辑分析可行实现路径。
- 不要在需求分析阶段直接开始改代码。
- 不明确的地方先列假设与影响，不要主观拍板。
