# 数据库操作技能

本技能用于配合业务开发完成数据库与 ES 相关操作，并把数据库改动沉淀到交付文档中。

## 适用场景

- 查询 MySQL / ES 数据
- 生成测试数据
- 生成插入、更新、删除、迁移脚本
- 生成建表、加字段、改字段 SQL
- 为发布说明整理数据库变更摘要

## 关键规则

- 修改前先查询确认影响范围。
- 删除、更新、建表、改表结构等高风险操作前必须再次确认范围。
- 不要在 skill 文本中硬编码仓库内凭据；优先使用本地安全配置、现有脚本或环境变量。
- 重要数据库改动必须同步写入 `.workflow/change-summary.json` 和 `.workflow/release-notes.md`。
- 如果只是生成脚本而未执行，要明确标注“未执行，仅生成”。

## 输出要求

- 如果本次需求涉及数据库改动，至少输出以下之一：
  - SQL 脚本
  - 数据迁移脚本
  - 测试数据脚本
  - 数据库变更摘要

## 建议步骤

1. 查实体、Mapper、XML 或 ES Model，确认真实结构。
2. 先生成查询语句或校验语句确认范围。
3. 再生成变更 SQL / DSL / 测试数据脚本。
4. 将表名、字段、索引、影响范围、执行建议整理进发布说明。
