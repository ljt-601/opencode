# Excel 文本入表

按以下流程处理：

1. 先确认目标 Excel 文件路径；未指定时，默认使用 `/Users/bryle/Documents/截图复刻表格.xlsx`。只有当用户明确指定其他文件时，才切换目标表格。
2. 将用户粘贴的原始文本按“公司名 + 明细行 + 可选成本行”拆成记录块。
3. 明细行优先按 `经手人+部门+金额（备注）` 结构解析。
4. 识别字段并写入列：
   - A：序号，按最后一条记录递增
   - B：日期，未提供时默认当天
   - C：部门
   - D：经手人
   - E：客户名称
   - F：金额
   - G：硬成本
   - H：利润，有硬成本时用 `金额-硬成本`，没有硬成本时默认等于金额
   - I：回款业务类型，优先根据备注或文本判断；包含“续费/续签/地址续费”默认写“续签”，否则默认“新签”
   - J：记账期，文本未提供时留空
   - K：备注，保留括号内信息或补充说明
   - L：合同是否交，文本未提供时留空
5. 追加写入时复制上一行样式，保持边框、填充、字体、对齐方式一致。
6. 写入后抽样校验新增行，至少确认：序号、日期、金额、成本、利润。

## 推荐执行方式

优先调用脚本：`scripts/import_text_to_excel.py`

示例：

默认目标表格：`/Users/bryle/Documents/截图复刻表格.xlsx`

```bash
python3 "/Users/bryle/.config/opencode/skills/excel-text-importer/scripts/import_text_to_excel.py" \
  --excel "/Users/bryle/Documents/截图复刻表格.xlsx" \
  --text-file "/tmp/import.txt"
```

如果用户直接在对话里给出文本：
1. 未指定 Excel 时，直接使用 `/Users/bryle/Documents/截图复刻表格.xlsx`
2. 先把文本写入临时文件
3. 再执行脚本
4. 将脚本输出的新增行结果反馈给用户

## 解析规则

- 公司名：默认是每个记录块第一行
- 明细行：默认是每个记录块第二行
- 成本行：若存在 `成本123`、`硬成本123`、`成本：123` 这类写法，则识别为成本
- 备注：取明细行括号内内容，如 `800（地址续费+二维码）`
- 金额：取明细行中的数字金额
- 日期：若原始文本没给日期，统一写当天

## 注意事项

- 修改 Excel 前先读取或检查目标文件是否存在
- 不要覆盖旧数据，只做追加
- 如果解析不出金额、经手人或部门，要先向用户确认，不要猜
- 如果同一批文本结构混乱，先把可确定部分导入，再列出异常行
